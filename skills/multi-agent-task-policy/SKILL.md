# 멀티 에이전트 작업 정책

아래 3가지는 예외 없이 적용한다.

## 1) Task 종료 규칙

각 개발자는 본인 Task를 끝낼 때 반드시 다음 순서를 따른다.

1. 담당 영역 기준 빌드/테스트 실행
2. 결과 확인 후 관련 파일만 커밋
3. `git push origin <현재 브랜치>`

검증이 실패하면 push하지 않는다.

## 2) worktree/branch 분리 규칙

각 개발자는 독립된 `git worktree`와 전용 브랜치를 사용한다.

- 한 worktree에는 한 개발자만 작업한다.
- 브랜치는 Task 단위로 분리한다.
- 다른 개발자 worktree에서 커밋/푸시하지 않는다.

## 3) 담당 영역 분리 규칙

프론트엔드/백엔드 개발자는 본인 영역만 수정한다.

- 프론트엔드 담당: `frontend/**`
- 백엔드 담당: `backend/**`
- 공통 문서/계약 변경이 필요하면 변경 이유와 영향 범위를 함께 기록한다.

## 출력 형식

```md
## 멀티 에이전트 정책 점검
- 담당 영역:
- worktree/branch:
- 빌드/테스트 결과:
- push 결과:
- 영역 외 변경 여부:
```
