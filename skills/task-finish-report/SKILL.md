# 작업 마무리 보고

작업 완료 시 아래 순서를 지킨다.

1. 현재 브랜치와 변경 파일을 확인한다.
- `git branch --show-current`
- `git status --short`

2. 변경 성격에 맞는 검증을 실행한다.
- 코드 변경: 빌드/테스트 실행
- 문서만 변경: 테스트 생략 가능
 - 이 저장소에서는 `project-validation-gate` 기준을 따른다.

3. 작업 관련 파일만 스테이징한다.
- 무관 파일은 포함하지 않는다.

4. 커밋 메시지를 규칙에 맞게 작성한다.
- 형식: `<type>(<scope>): <summary>`

5. 원격에 푸시하고 결과를 기록한다.
- `git push origin <current-branch>`

6. 최종 보고를 남긴다.
- 브랜치, 커밋 해시, 검증 결과, 잔여 이슈 포함
- API 계약을 변경했다면 README API 목록 동기화 여부를 포함한다.

## 출력 형식

```md
## 작업 완료 보고
- 브랜치:
- 커밋:
- 변경 파일:
- 검증 실행/결과:
- 푸시 결과:
- 잔여 이슈:
```
