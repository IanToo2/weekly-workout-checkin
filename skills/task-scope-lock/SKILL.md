# 작업 범위 고정

아래 순서로 시작한다.

1. 작업 목표를 한 줄로 확정한다.
- 형식: `목표: <무엇을 왜 바꾸는지>`

2. 담당 영역을 명시한다.
- `frontend` 또는 `backend` 중 하나를 선택
- 선택한 영역 외 변경은 원칙적으로 금지

3. 작업 worktree/브랜치를 선언한다.
- 형식: `worktree: <경로> | branch: <브랜치명>`

4. 변경 범위를 명시한다.
- 형식: `수정 대상: <파일/폴더 목록>`
- 목록 밖 파일은 수정하지 않는다.

5. 완료 조건(DoD)을 2~5개로 작성한다.
- 테스트/동작/출력 기준으로 검증 가능하게 작성

6. 제외 범위를 명시한다.
- 형식: `이번 작업에서 하지 않음: <항목>`

## 출력 형식

```md
## 작업 선언
- 목표:
- 담당 영역:
- worktree/branch:
- 수정 대상:
- 완료 조건:
- 이번 작업에서 하지 않음:
```
