# 작업 범위 고정

아래 순서로 시작한다.

1. 작업 목표를 한 줄로 확정한다.
- 형식: `목표: <무엇을 왜 바꾸는지>`

2. 변경 범위를 명시한다.
- 형식: `수정 대상: <파일/폴더 목록>`
- 목록 밖 파일은 수정하지 않는다.

3. 완료 조건(DoD)을 2~5개로 작성한다.
- 테스트/동작/출력 기준으로 검증 가능하게 적는다.

4. 제외 범위를 명시한다.
- 형식: `이번 작업에서 하지 않음: <항목>`

5. 착수 전에 리스크를 한 줄씩 기록한다.
- 형식: `리스크: <문제> / 대응: <방법>`

## 출력 형식

아래 템플릿으로 응답한다.

```md
## 작업 선언
- 목표:
- 수정 대상:
- 완료 조건:
- 이번 작업에서 하지 않음:
- 리스크/대응:
```
