# Skills 운영 규칙 (v2)

1. 스킬 하나는 작업 하나만 다룬다.
- 범위를 좁고 명확하게 유지한다.

2. 모든 스킬은 `SKILL.md`를 필수로 가진다.
- frontmatter는 `name`, `description`만 사용한다.

3. 지시문은 짧고 실행 가능하게 작성한다.
- 파일 경로, 명령, 검증 조건을 우선한다.

4. 반복되는 경우에만 리소스를 추가한다.
- `scripts/`: 반복 자동화
- `references/`: 필요 시 로드하는 참고 정보
- `assets/`: 템플릿/산출물 파일

5. 멀티 에이전트 기본 정책을 고정한다.
- 각 Task 종료 시 빌드/테스트 후 `git push`한다.
- 각 개발자는 `git worktree`로 분리된 작업 디렉터리와 브랜치를 사용한다.
- 프론트엔드/백엔드 개발자는 본인 담당 영역만 수정한다.

6. description에 트리거를 명확히 적는다.
- 무엇을 하고, 언제 쓰는지 함께 기술한다.

7. 역할군보다 작업 결과 중심으로 스킬을 설계한다.

8. 스킬은 같은 폴더에서 갱신하고, 분리가 필요할 때만 새로 만든다.

9. 컨텍스트 비용을 줄인다.
- `SKILL.md`는 간결하게 유지한다.
- 상세 내용은 필요 시 참조 파일로 분리한다.
