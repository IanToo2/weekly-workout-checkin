# Skills 재구축 규칙 (v1)

1. 스킬 하나에 작업 하나만 담는다.
- 범위를 좁고 명확하게 유지한다.

2. 모든 스킬은 `SKILL.md`를 필수로 가진다.
- YAML frontmatter는 `name`, `description`만 사용한다.

3. 지시문은 짧고 실행 가능하게 작성한다.
- 추상 설명보다 파일 경로, 명령, 검증 조건을 우선한다.

4. 반복 작업일 때만 리소스를 추가한다.
- `scripts/`: 반복되는 확정 절차 자동화
- `references/`: 필요 시 불러올 도메인 정보
- `assets/`: 템플릿/산출물 파일

5. 활성화 전 품질 기준을 확인한다.
- 이름은 소문자/숫자/하이픈만 사용한다.
- 트리거 문구가 명확한지 확인한다.
- 중복되거나 오래된 스킬은 제거한다.

6. description에 트리거를 명시한다.
- 무엇을 하는지, 언제 쓰는지를 함께 적는다.

7. 기본 원칙은 역할군이 아닌 결과 중심이다.
- 페르소나보다 작업 결과 단위로 스킬을 나눈다.

8. 버전은 같은 폴더에서 갱신한다.
- 분리가 명확히 필요할 때만 새 스킬로 분리한다.

9. 컨텍스트 비용을 최소화한다.
- `SKILL.md`는 간결하게 유지한다.
- 상세 내용은 선택 리소스로 분리한다.

10. 신규 스킬은 아래 순서로 만든다.
- 사용자 예시 확인
- 재사용 요소 계획
- 생성/수정
- 검증
- 반복 개선
