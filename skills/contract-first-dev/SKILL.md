# 계약 우선 구현

구현 전에 계약을 먼저 고정한다.

1. 계약 변경 여부를 먼저 판정한다.
- 대상: API 경로, 요청/응답 필드, 타입, DB 스키마
- 결과를 `변경 없음` 또는 `변경 있음`으로 명시한다.

2. 계약을 텍스트로 확정한다.
- 엔드포인트/필드/타입/필수값/에러 형식을 표로 정리한다.

3. 변경 영향 범위를 지정한다.
- 백엔드: 컨트롤러/서비스/리포지토리
- 프론트: API 호출부/타입/UI 상태 처리
- 문서: README 또는 계약 문서

4. 구현은 계약 기준으로 진행한다.
- 계약과 다르게 구현하지 않는다.
- 불일치 발견 시 먼저 계약을 갱신하고 구현한다.

5. 종료 시 동기화 상태를 보고한다.
- 코드, 타입, 문서가 동일 계약을 따르는지 확인한다.

## 출력 형식

```md
## 계약 점검 결과
- 계약 변경 여부:
- 확정 계약 요약:
- 영향 범위:
- 구현 반영 상태:
- 문서 반영 상태:
- 잔여 리스크:
```
